## Source Data (raw, in `data/`)

### Apple Music

| File | Description |
|------|-------------|
| `Apple Music - Play History Daily Tracks.csv` | **Primary source.** One row per track × day. |
| `Apple Music Play Activity.csv` | 177MB. Per-play events with Album Name. Used for album sessions. |

**Daily Tracks columns used:**

| Column | Type | Description |
|--------|------|-------------|
| `Track Description` | str | `"Artist - Song"` format. Split into artist + song. |
| `Date Played` | int | `YYYYMMDD`. Parsed to date. |
| `Hours` | str | Hour(s) of day, local time. Can be multi-value: `"14, 18, 19"`. |
| `Play Duration Milliseconds` | int | Duration actually played. Converted to minutes. |
| `Play Count` | int | Plays that day (pre-aggregated by Apple). |
| `Source Type` | str | Device: `IPHONE`, `MACINTOSH`, `IPAD`, `APPLETV`. |
| `Country` | str | Country of playback. Used for timezone context. |

**Play Activity columns used:**

| Column | Type | Description |
|--------|------|-------------|
| `Song Name` | str | Track name (no artist). |
| `Album Name` | str | Album name. Key for album sessions. |
| `Event Start Timestamp` | str | ISO timestamp (UTC). Truncated to day. |
| `Play Duration Milliseconds` | float | Filter: ≥30s = real play. |

### Spotify

| File | Description |
|------|-------------|
| `spotify/Streaming_History_Audio_*.json` | Extended streaming history. One entry per play. |

**Columns used:**

| Column | Type | Description |
|--------|------|-------------|
| `ts` | str | ISO timestamp (UTC). Converted to local time via `conn_country`. |
| `conn_country` | str | Country of connection. Used for UTC→local conversion. |
| `master_metadata_album_artist_name` | str | Artist. |
| `master_metadata_track_name` | str | Song. |
| `master_metadata_album_album_name` | str | Album. Used for album sessions. |
| `ms_played` | int | Milliseconds played. Filter: ≥30s. |

**Timezone handling:** Spotify timestamps are UTC. Converted to local time using `conn_country` offset (CO=-5, ES=+1, etc). Apple Music `Hours` field is already local time. This means hour-of-day patterns reflect actual daily routine wherever the user was.

**Unused but available:** `platform`, `shuffle`, `reason_start`, `reason_end`, `spotify_track_uri`, `skipped`.

## Processed Data (in `app/public/data/`, generated by `process.py`)

### artist-month.json

**Grain:** artist × month. **Key:** `a` + `m`. **Core table** — most analytics derive from it.

| Field | Type | Description |
|-------|------|-------------|
| `a` | str | Artist name |
| `m` | str | Month `"YYYY-MM"` |
| `h` | float | Hours listened |
| `p` | int | Play count |
| `s` | int | Unique songs played |

**Used by:** StatsRow, MonthlyTrend, TopArtists, ArtistDetail, GenreChart, CountryChart, DecadeChart, ScoreCard, all comparison modes.

### songs.json

**Grain:** artist × song. **Key:** `a` + `s`.

| Field | Type | Description |
|-------|------|-------------|
| `a` | str | Artist name |
| `s` | str | Song name |
| `pm` | object | `{"YYYY-MM": [plays, minutes]}` per month |

**Used by:** TopSongsTable, ArtistDetail (comparison), ResurfaceSongs (recuperar).

### album-sessions.json

**Grain:** artist × album × day (only days with 3+ unique tracks). **Key:** `a` + `al` + `d`.

| Field | Type | Description |
|-------|------|-------------|
| `a` | str | Artist name |
| `al` | str | Album name (case-normalised, deduplicated) |
| `d` | str | Date `"YYYY-MM-DD"` |
| `m` | str | Month `"YYYY-MM"` |
| `t` | int | Unique tracks played that day |

**Used by:** TopAlbumsChart. Score = `sum(t)` for qualifying sessions. Sources: Spotify (album direct), Apple Music Play Activity (album→artist via song overlap + manual overrides).

**Album→artist resolution:** Spotify data has artist+album together (reliable). Apple Music Play Activity has album but no artist — resolved by matching song names against Daily Tracks artist lookup, weighted by Spotify votes (3×) vs Apple votes (1×). Manual overrides for known mismatches (Beyoncé, Kendrick Lamar, Oasis covers, etc).

### loops.json

**Grain:** artist × song × day (only days with play count > 1). **Key:** `a` + `s` + `d`.

| Field | Type | Description |
|-------|------|-------------|
| `a` | str | Artist name |
| `s` | str | Song name |
| `d` | str | Date `"YYYY-MM-DD"` |
| `m` | str | Month `"YYYY-MM"` |
| `p` | int | Play count that day |

**Used by:** LoopsChart. For Spotify: grouped by (artist, song, day) and counted. For Apple Music: `Play Count` field already >1. Dynamic filter buttons generated from data (3x to max).

### artist-genres.json

**Grain:** artist. **Key:** `a`. **Source:** MusicBrainz (via `artists-enriched.json`).

| Field | Type | Description |
|-------|------|-------------|
| `a` | str | Artist name |
| `g` | str[] | Genre tags (max 3, filtered for non-genres, with manual overrides) |
| `country` | str\|null | Country code or city |
| `type` | str\|null | `"Group"` or `"Person"` |
| `begin` | int\|null | Year artist started |

**Used by:** GenreChart, CountryChart, DecadeChart. Genre/country filters in FilterContext use `artistMeta` (loaded from this file) to filter all dashboard sections.

### artist-similar.json

**Grain:** artist → similar artists. **Key:** artist name. **Source:** Last.fm.

| Field | Type | Description |
|-------|------|-------------|
| (key) | str | Artist name |
| (value) | str[] | Up to 10 similar artists |

**Used by:** TopArtistsChart (when artist selected, shows "Similares a X" panel).

### artist-scores.json

**Grain:** artist → review score. **Key:** artist name. **Source:** `artist-reviews.json`.

| Field | Type | Description |
|-------|------|-------------|
| (key) | str | Artist name |
| (value) | float | Average review score (0-10) across Pitchfork/NME/RA/Uncut |

**Used by:** ScoreCard (year heatmap in general view, single score when artist/genre/country selected).

### discover.json

**Grain:** recommended artist. Generated by standalone Python script.

| Field | Type | Description |
|-------|------|-------------|
| `artist` | str | Recommended artist name |
| `d1` | float | Direct similarity score (0-1, normalised) |
| `d2` | float | 2nd degree similarity score |
| `d3` | float | Influence chain score |
| `d4` | float | Genre affinity score |
| `because` | str[] | Top 5 source artists (why this recommendation) |
| `tags` | str[] | Genre tags |
| `listeners` | int | Last.fm global listener count |
| `country` | str\|null | Country |
| `myPlays` | int | How many times user has played this artist (0 = never heard) |

**Used by:** DiscoverDashboard (/descubrir). Frontend applies weighted scoring via presets (mix/familiar/explorar/sorprender) and filters (tags, seeds, exclude, familiarity, min score).

**Four dimensions:**
- **d1 (direct):** `Σ log(1 + hours_source)` for each listened artist whose Last.fm similar includes this candidate
- **d2 (2nd degree):** Similar of similar, dampened ×0.3. Requires expanded Last.fm data.
- **d3 (influence):** Wikidata influence chains. Influenced-by ×1.5, influenced ×0.8.
- **d4 (genre):** Genre profile overlap (listened genre hours / total vs candidate tags)

### weekday-month.json, source-month.json, hourly-month.json

Supporting tables for WeekdayChart, HourlyChart. weekday-month and source-month have artist-level data (filterable by artist/genre/country). hourly-month is hour × month only (no artist dimension).

### resurface.json

Precomputed. Used as fallback in /recuperar. The slider mode computes live from artist-month + songs.

## Enrichment Data (in `data/`)

All keyed by **artist name** (string match).

### artists-enriched.json — MusicBrainz

| Field | Type | Coverage |
|-------|------|----------|
| `artist` | str | 100% |
| `mbid` | str\|null | ~75% |
| `tags` | str[] | ~56% |
| `country` | str\|null | ~70% |
| `type` | str\|null | ~75% |
| `beginYear` | int\|null | ~55% |
| `score` | int\|null | ~75% (search match confidence) |

### artists-lastfm.json — Last.fm (base + expanded)

| Field | Type | Coverage |
|-------|------|----------|
| `artist` | str | 100% of base + 2nd/3rd degree |
| `tags` | str[] | ~90% |
| `similar` | str[] | ~90% (up to 10 per artist) |
| `listeners` | int | ~94% |
| `playcount` | int | ~94% |

Base: 14.3k listened artists. Expanded via `expand_lastfm.py`: fetches similar data for artists referenced as "similar" but not in base. Multiple passes for 2nd + 3rd degree. Currently ~76k total.

### artists-wikidata.json — Wikidata

| Field | Type | Coverage |
|-------|------|----------|
| `artist` | str | 100% |
| `genres` | str[] | ~45% |
| `influences` | str[] | ~15% (unique to this source) |
| `country` | str\|null | sparse |

### artists-discogs.json — Discogs (in progress)

| Field | Type | Coverage |
|-------|------|----------|
| `artist` | str | ~100% (14.3k) |
| `discogsId` | int\|null | ~89% of processed |
| `genres` | str[] | ~89% of processed |
| `styles` | str[] | ~88% of processed |
| `releases` | array | `{title, year, genres, styles}` per release |

### reviews-unified.json — Merged Reviews

| Field | Type | Description |
|-------|------|-------------|
| `artist` | str | Artist name |
| `album` | str | Album name |
| `score` | float | Normalised 0-10 |
| `date` | str | Publication date |
| `source` | str | `pitchfork`, `nme`, `ra`, `uncut` |
| `url` | str | Review URL |

~42.6k reviews. Pitchfork (0-10 native), NME (×2), RA (8.0/6.0 from recommended), Uncut (0-10 native).

### artist-reviews.json — Matched Reviews ↔ Listening

| Field | Type | Description |
|-------|------|-------------|
| `artist` | str | Key (matches listening history) |
| `hours` | float | Total hours listened |
| `avgScore` | float\|null | Weighted average review score |
| `reviewCount` | int | Number of reviews |
| `sources` | str[] | Which publications |
| `albums` | array | Per-album: `{album, score, source, date}` |

4,061 matched artists = 66% of listening hours. Avg score 7.4.

## Shared UI Components

### sharedColors.ts

Consistent colour assignment for items appearing in multiple comparison columns. Each shared item gets a unique color from 8-color palette (rose, sky, amber, emerald, violet, fuchsia, cyan, orange). Non-shared items use neutral grey. Used by: TopSongsTable, LoopsChart, TopAlbumsChart, ArtistDetail (comparison mode).

### FilterContext.tsx

Global state: `timeRange` (all / years[] / month), `selectedArtist`, `selectedGenre`, `selectedCountry`. Derives `FilteredData` with `topArtists`, `topSongs`, `activeArtists` (for genre/country cascading), `isComparing`, `comparisonYears`. All dashboard components consume this.

## Join Keys & Known Issues

- All tables join on **artist name** (string). No universal ID across sources.
- Case sensitivity handled via case-insensitive + normalised matching in `match_reviews.py`
- Name variants: `"ROSALÍA"` / `"Rosalía"` normalised in `process.py`. `"Kanye West"` → `"Ye"` via MusicBrainz alias search.
- Album→artist: Spotify reliable, Apple Music via heuristic + overrides.
- Album name deduplication: case-insensitive, keeps most common casing.
- MusicBrainz tags: filtered for non-genres + manual overrides for known misclassifications.
- Hourly data: Apple Music multi-value hours split client-side. Spotify UTC→local via `conn_country`.

## Data Flow

```
1. Raw exports (Apple Music CSV + Spotify JSON)
         ↓
2. process.py → 8 JSON files for frontend
         ↓
3. Frontend loads JSONs, computes filters/aggregations client-side
         ↓
4. Enrichment scripts (MB, Last.fm, expand_lastfm, Wikidata, Discogs) → per-source JSON
         ↓
5. process.py reads artists-enriched.json → artist-genres.json for frontend
         ↓
6. Scrape scripts (Pitchfork, NME, RA, Uncut) → per-year review JSONs
         ↓
7. merge_reviews.py → reviews-unified.json
         ↓
8. match_reviews.py → artist-reviews.json → artist-scores.json
         ↓
9. Discovery engine (Python) → discover.json (4-dimensional scores)
         ↓
10. update.py orchestrates all of the above
```
